1 - Introduction

DINO is designed and implemented as a tool to use GAP for field daily production optimization workflow. It is an interface to simplify and enhance user exprience and efficiency using Integrated Network Model (INM) for near real-time field and well optimization. In addition, DINO performs calibration of GAP model on frequent basis.

DINO was built using Python web framework and Petex OpenServer API for automating calculations in IPM software products.

2 - Step by step guide

2.1 - Setting up line-up and performing GAP calculation


2.1.1 - Go to Setup page. Here you can setup parameters prior to calculations.



2.1.2 - Change Qgas max and Separator pressures for KPC and Unit-2



2.1.3 - Change well route by selecting available routes from dropdown list



2.1.4 - Shut 1 well by setting target FWHP to -1



2.1.5 - Sort wells by GOR. Click twice on GOR label of the table



2.1.6 - Fix well by checking "Fixed Target FWHP" checkbox



2.1.7 - Save State for calculation



2.1.8 - Press OK



2.1.9 - Go to GAP calculation



2.1.10 - Press Start to start calculation



2.1.11 - Calculation logging



2.1.12 - Calculation complete. Constraints that exceed the limits are highlighted with yellow.



2.1.13 - Go to Results page



2.1.14 - See well results of wells configured in Setup



2.1.15 - Save current results as reference case for next calculations for comparison



2.1.16 - Press OK to proceed saving.





2.1.17 - Go to Setup and modify line-up, change well route and uncheck fixed well.





2.1.18 - Save State, perform GAP calculation and go to Results. You can now compare current results against previously saved ref case



2.1.19 - You can also delete reference case results.
